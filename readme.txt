2.1.0升级到2.2.0的方法:
升级前先备份你网站上的 templates\default\html\index.htm admin/config.php两个文件(或则你觉得需要备份的模板文件),loadsystem目录里面的文件。
1)二进制方式上传所有文件.
2)运行admin/update210-220.php
3)升级后，需要重新设置一下您的后台的基本选项（比如系统所在地址）。
4)拷贝备份的templates\default\html\index.htm文件,按照备份的admin/config.php重新配置你的数据库.根据备份的loadsystem目录里面的文件,重新配置你的整合的论坛数据库.
5)设置你的默认整合的论坛(详细见后台)

2.0.6升级到2.1.0的方法:
升级前先备份你网站上的 index.php admin/config.php两个文件
1)二进制方式上传所有文件(除了loadsystem里面的文件).
2)运行admin/update206-210.php
3)升级后，需要重新设置一下您的后台的基本选项（比如系统所在地址）。
4)拷贝备份的index.php里面的标签到index.htm模板。(注意，这些首页调用全部都可以用新功能“标签”来替换，这样能够提高首页生成效率)

2.0或2.01升级到2.0.1 special的方法：
1) 上传所有文件,覆盖原来的文件.zend的文件需要FTP二进制上传(比如mkarticle.php,mksort.php,functions.php)。
2) 配置你的数据库文件admin/config.php
3) 如果你之前已经安装了niuboy的前期html版本，运行admin/update2html_2.php,否则,运行admin/update2html.php
4) 升级后，需要重新设置一下您的后台的基本选项（比如系统所在地址）。
5) 如果你的空间是保护模式，你需要事先生成好html所有目录，设置属性为777
6) zend optimizer 必须安装
////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
使用原来的模板：
1) 导出原来的模板，保存在tools目录里面
2) 然后，把special版本 admin目录下的模版导入，覆盖原来模版。
3) 运行tools/convertstyle.php
4) 导入转换后的模板文件。
5) 阅读文件"模板手动修改说明.txt"，完成里面要求的模板修改。
////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
如果你想恢复为原来的2.0.1版本:
1) 用pa2.0.1版本覆盖特别版。
2) 修改mysql数据库配置文件
3) 替换2.0.1的风格

安装方法：
运行admin/install.php
////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
使用说明:
1) 如果您添加了文章，需要进入分类html管理，对你添加了文章的分类，进行生成。 
2) 如果您添加了新的分类，您需要在生成静态数据前，先刷新一下缓冲。
3) htmldata 目录 和系统主目录需要设置为 0777 属性（可读写，可以用ftp登陆后修改）

////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
首页调用说明：

showarticle($sortid,$templatename,$main_article ,$main_len );//一般文章调用 参数(分类id,"模板名",文章条数,标题长度)

showimgarticle($sortid,$templatename,$img_article,$img_article_len);//图片文章调用  参数(分类id,"模板名",图片文章条数,标题，

摘要分别的长度)


举例:
可以使用默认值，调用方法如下，（调用分类id为3，模板名为  myart,一般文章文类,模板变量为$showart,假设模板已经做好，如何做请参考

上文原始版本的MOD）

1) 在$phparticleurl/index.php
在 require "global.php";后面添加代码

$showart = showarticle(3,"myart",$main_article ,$main_len );也就是说添加了后台变量之后，后两个参数可以不用数字代替，使用默认

就可以。


如果你想让某个栏目的文章数和标题长度不一样，那么可以这么做（还是用上面的例子，假设文章数要变成20，标

题长度变成50的话）

$showart = showarticle(3,"myart",20,50);


图片文章的调用参数含义一样的：
$showimgart = showimgarticle(2,"imgart",$img_article,$img_article_len );

注意：
分类id请自己用你要调用的分类的编号代替
如果有多个分类调用
那么请添加多次


2) 首页模板的修改

在模板  home 你想显示分类文章的地方添加
<table width="96%" border="0" cellpadding="0" cellspacing="0">
         $showart
</table>

如果是图片文章，
在 home 模板中显示图片的位置添加变量
$showimgart
